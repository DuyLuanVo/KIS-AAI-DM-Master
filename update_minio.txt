Bước 1: Tích hợp MinIO & Di chuyển dữ liệu ảnh (Object Storage Migration)
Mục tiêu: Loại bỏ lưu trữ ảnh local, chuyển toàn bộ keyframes lên MinIO.
Công việc cụ thể:
Cập nhật file Docker Compose của dự án, thêm service minio và mc (MinIO Client).
Tạo bucket tên là kis-keyframes trên MinIO. Cấu hình policy của bucket này là Public Read-Only để cho phép tải trực tiếp (hoặc cấu hình private nếu muốn dùng cơ chế sinh Pre-signed URL an toàn).
Viết script Python trong 

scripts
 đọc toàn bộ thư mục ảnh cục bộ data/Keyframes_LXX và upload lên bucket của MinIO bằng thư viện boto3 hoặc minio.
📌 Bước 2: Cập nhật Pipeline nạp dữ liệu Qdrant (Qdrant Payload Update)
Mục tiêu: Cập nhật payload của vector trong Qdrant trỏ tới MinIO thay vì file local.
Công việc cụ thể:
Chỉnh sửa file cấu hình 

pipeline/config.py
 để khai báo các biến môi trường cho MinIO (Endpoint, Bucket, Credentials).
Cập nhật script nạp dữ liệu 

pipeline/loaders/run_loader.py
: Sửa trường "jpg_path" trong cấu trúc dữ liệu gửi lên Qdrant thành Object Key tương đối trên MinIO (Ví dụ: keyframes/L21_V001/001.jpg).
Chạy lại loader để cập nhật toàn bộ database Qdrant.
📌 Bước 3: Cấu hình API Backend & Frontend để hiển thị ảnh từ MinIO
Mục tiêu: Frontend và Backend đọc ảnh trực tiếp từ MinIO qua URL bảo mật.
Công việc cụ thể:
Cập nhật 

backend/app/config.py
 để nạp cấu hình MinIO.
Tại API Search của Backend, khi lấy danh sách kết quả từ Qdrant, sử dụng thư viện boto3 để sinh Pre-signed URL cho từng ảnh rồi gán vào trường image_url gửi về cho Frontend.
Cập nhật Frontend hiển thị trực tiếp ảnh bằng image_url nhận được từ API. Lúc này, ta có thể khai tử server static local 

frontend/serve.py
.
